# AI Risk Assessment Checklist for CSOs

*Part of the "AI Literacy for NGOs: Starter Kit" — Cvetanichin Consultancy*

Run every new AI tool through this checklist before adopting it organisation-wide.

## Data Privacy

| Question | Yes / No | Notes |
|---|---|---|
| Does the tool store my data on third-party servers? | ☐ | If yes, can I opt out? |
| Does the tool comply with GDPR? | ☐ | Check the provider's privacy policy directly. |
| Will I input sensitive data (PII, confidential donor or beneficiary records)? | ☐ | If yes, avoid this tool or use a privacy-focused alternative. |

## Bias

| Question | Yes / No | Notes |
|---|---|---|
| Has the tool been tested for bias relevant to my sector? | ☐ | Look for third-party audits or published evaluations. |
| Can I manually review AI outputs before they influence a decision? | ☐ | Always double-check anything affecting funding or beneficiary status. |

## Over-Reliance

| Question | Yes / No | Notes |
|---|---|---|
| Will this tool be used for critical decisions (grant allocations, beneficiary selection)? | ☐ | If yes, a human review step is mandatory. |
| Do I have a manual backup process if the tool fails or is unavailable? | ☐ | E.g. a manual screening process for grant calls. |

## Ethics & Governance

| Question | Yes / No | Notes |
|---|---|---|
| Does the tool align with my NGO's stated values? | ☐ | E.g. avoid tools with exploitative data practices. |
| Have I consulted relevant staff before adopting this tool? | ☐ | Get buy-in, not just sign-off. |

## Rule of Thumb

If you answer **"No" to any of the first three data-privacy questions**, do not use the tool — find a privacy-focused alternative or restrict its use to non-sensitive tasks only.
